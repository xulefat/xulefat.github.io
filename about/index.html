我有一个坏的习惯——很少去记录东西，这是很致命的。

东西多了，时间久了，知识就慢慢的模糊。 越是模糊，越是说不清楚，这对一个创业者来说，无疑等于一个“文盲”

★动机:为什么搭建本站点？

◇记录与分享

&emsp;&emsp; 记录，把东西记录下来。人总是会忘记一些学过的东西。那么对个人而言，记录至少有以下几个作用。1，加深记忆，延迟淡忘的时间节点。因为在记录的过程中，其实算得上是把东西回顾一了遍。2，备稿。在记录的过程中，我是以我的理解去记录的，这相当于提前备稿了，这有利于应对突然袭来的提问。当别人提问时，讲不出来话，这对于一个创业者来说是很致命的，讲不出来，表面上看起来像一个文盲。3，建立一个外脑。当需要的时候，外脑可以快速检索，帮助我快速唤起某些东西。


&emsp;&emsp; 分享，把东西分享出去。人们可能会通过一下“标签“去初步认识一个人。那么对个人而言，这个站点算是我的一个“标签“吧。当然，我在学习中也利用了不少的互联网资源，如今我把我的一些思路，一些方法分享出来，希望能给大家一些帮助。

★记录什么？

&emsp;&emsp;本站点主要记录一些我对创业的一些思考和我的创业历程。

★本站点的搭建历程

◇我的建站经历。

  • 我会一点html，css，js，linux，以及其他的一些技术。半年前（2022年12月的时候吧）成功用WordPress＋阿里云方案建站。但是并没有将该站点运营。
  • 本站于2023年5月14日搭建成功。用的是Hoxe＋github，主题用的是next。这次要好好写博客。

◇本站点的搭建过程。

&emsp;&emsp; 因为我在网上找的教程大都为老版本的教程，我在搭建本站时也遇到了一下问题，所以我觉得有必要将Hexo＋github＋next方案建站的大概流程梳理一下，然后列出我建站时遇到的一些注意事项。
&emsp;&emsp;&emsp; Hexo本地部署。
&emsp;&emsp;&emsp; 第一步。分一个新盘符（我分出的是E盘），将安装git和node.js安装到E盘。
&emsp;&emsp;&emsp; 第二步。在E盘新建一个文件夹，命名为blog，在blog文件夹新建一个myblog。（文件夹名字随便命名也没问题。）
&emsp;&emsp;&emsp; 第三步。点进去myblog文件夹中右击，点击git bash here，进入git命令行窗口。分别在命令行窗口输入命令。
&emsp;&emsp;&emsp; 第四步。在窗口输入hexo s（开启本地hexo服务的命令），负责地址，在浏览器中打开，有个地球就算是本地部署成功了。
注意:在窗口输入ctrl c或者退出窗口会关闭本地hexo服务。这样会导致在本地预览不到网站。
&emsp;&emsp;&emsp; 把Heox部署到github。
&emsp;&emsp;&emsp; 第一步。注册github账号，然后创建仓库。仓库名字格式为“你的用户名+github.io”就行。
&emsp;&emsp;&emsp; 第二步。生成秘钥，然后再github中添加秘钥。执行命令git config --global user.name "yourname"之后执行命令git config --global user.email "youremail" 之后执行命令ssh-keygen -t rsa -C "youremail"之后你就会得到一个路径，这个路径就是存放密钥的路径，然后进去，点开名字带pub那个文件并复制那个文件的内容。进入GitHub，点击setting，找到ssh and ghg key，把复制的内容粘贴到下面大的那个框，上面小的那个框随便输入个名字就行，最后点击add key。
&emsp;&emsp;&emsp; 第三步。修改站点配置文件。点进去myblog，点进去config.yml文件。
&emsp;&emsp;&emsp; 第四步。将博客从本地推到github仓库。在blog目录下点开git bash，依次执行命令hexo clean，hexo g，npm install hexo-deployer-git --save，hexo d。
&emsp;&emsp;&emsp; 第五步。访问网站。等一分钟左右，上去GitHub仓库可以看到有文件，输入你的网址能看到文字和主题就算成功了。
注意:部署到github上后看不到样式。解决方法，把root改成root: /就好了。
。
&emsp;&emsp;&emsp; 主题安装
&emsp;&emsp;&emsp; 第一步。上hexo官网寻找主题，点击后进入GitHub页面，clone主题链接。在theme目录下打开git bash，输入命令git clone 主题链接。等待主题下完成。
&emsp;&emsp;&emsp;第二步。修改hexo全局配置config文件，将主题模块下原来的主题注释掉（不要删掉，是注释掉，23年5月8日，我直接删掉然后再配置新主题（theme： next），本地预览是空白页面。），然后按照格式把新主题的文件夹写上。
&emsp;&emsp;&emsp;第三步。分别执行命令hexo clean，hexo g，hexo s，本地预览成功。
&emsp;&emsp;&emsp;第四步。分别执行hexo clean，hexo g，hexo d，成功上传到GitHub仓库，网址访问成功。
&emsp;&emsp;&emsp;主题美化。
&emsp;&emsp;&emsp;状况:我在这里花了很多时间去美化，弄了一段时间后，突然发现，我的原型图好像不怎么好看了。而且网上的教程大都是老版本的。所以，就简单改了下图标颜色，原来改过的那些配置，大都改回来了。时机对的时候我再的博客升级吧。next的美化网上有很多很详细教程，这里我讲一下在我折腾的过程中一些要注意的地方吧。1，动态背景。8.16.0版本的next把内置的动态背景配置删掉了，改成内置的静态丝带背景。2，更改menu图标和更改社交图表和页脚图标（桃心）。首先把你需要的图标放到myblog\source\images文件夹下。然后这三个图标的更改都需要对hexo文件夹中的\source\ _data\styles.styl文件进行修改(没有的文件夹就新建)。向styles.styl文件夹输入如下代码/* sidebar social icon */ .guanyu { background-image: url('/images/bilibili.svg'); background-size: 1em 1em; opacity: 0.55; background-position: 0.05rem 0.2rem; background-repeat: no-repeat; height: 1rem; width: 1rem; border-radius: 0rem; /*鼠标停留在图标上时，图标呈现发光效果*/ &:hover { opacity: 1; } }guanyu是用来匹配图标的（匹配名）。url就是你图标的地址。然后进入主题配置文件
找到custom file paths，将style: source/_data/styles.styl的注释清除掉。最后就可以在主题配置文件中的|| fa fa-about中将图标改成你要的图标了（比如|| fa fa-about改成|| fa fa-custom guanyu，注意，custom一定要加上。）。
